一：创建密钥 链接本地与github 1.设置本地的ssh key，打开git bash,输入命令： ssh-keygen -t rsa -C "XXXXXX@XXXX.com" 其中双引号中是你注册github时用的邮箱。 一直回车，选择默认路径，和空密码。最后会在默认路径下生成.ssh文件夹，打开.ssh里面有两个文件，打开id_rsa.pub复制里面的密钥。 2：2.打开github,选择settings，点击ssh and gpg keys,选择ssh keys 右边的new ssh key。出现下面绿色框的内容，填写标题，并将自己刚才复制的密钥粘贴到key中。最后点击add ssh key. 3.查看是否成功。在git bash中输入命令： ssh -T git@github.com 会提示，是否continue，输入yes。后就会看到： Warning:Permanently added 'github.com,207.97.227.239' (RSA) to the list of known hosts. 　　Hi zhangsiyao11! You've successfully authenticated, but GitHub does not provide shell access. 这样就成功了，不用理会warning。 二：新建仓库,然后复制地址，用$ git remote add origin xxxxxx,如 1： git remote add origin https://github.com/tata859/demo.git 2: git push -u origin master 上传本地所在文件的所有代码和文档 3：如果出现错误： fatal: remote origin already exists 则执行以下语句： git remote rm origin 再执行git remote add origin https://github.com/tata859/demo.git即可。 三，也可以克隆刚才新建的仓库到本地，输入命令： git clone https://github.com/tata859/demo.git 后面的http是你的仓库的地址。 将你想要上传的项目文件复制到上面的目录 在此处打开git bash,输入如下命令： git init git add "文件名" git commit -m "文件名" 如果出现让你设置用户名和用户邮箱的提示，就按照提示上给的命令输入即可。后再次执行上面commit的命令。 git remote add origin https://github.com/tata859/demo.git http为你自己仓库的地址 如果出现错误： fatal: remote origin already exists 则执行以下语句： git remote rm origin 再执行git remote add origin https://github.com/tata859/demo.git即可。
